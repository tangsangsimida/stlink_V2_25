## 行为准则

本项目遵循[贡献者公约](https://www.contributor-covenant.org/version/2/0/code_of_conduct/)的行为准则。所有贡献者都应遵守此准则，共同营造一个友好、包容的社区环境。

## 如何贡献

### 报告Bug

如果您发现了Bug，请在GitHub上创建一个Issue，并包含以下信息：

1. 清晰的Bug描述
2. 复现步骤
3. 预期行为和实际行为
4. 环境信息（操作系统、软件版本等）
5. 相关日志或截图

### 提出功能建议

如果您有新的功能建议，请创建一个Issue，并包含以下信息：

1. 功能描述
2. 解决的问题或满足的需求
3. 可能的实现方案
4. 预期的用户界面或API

### 提交代码贡献

1. Fork本项目
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个Pull Request

## 文档规范

1. 所有公共API都需要文档注释
2. 使用Doxygen风格的注释
3. README.md需要保持最新
4. 重要设计决策需要在文档中说明

## 测试要求

1. 所有新功能都需要相应的单元测试
2. 修复Bug时需要添加测试用例
3. 确保所有测试在主要支持的平台上通过
4. 性能相关的修改需要提供基准测试结果

## 提交Pull Request

在提交Pull Request之前，请确保：

1. 代码遵循项目的编码规范
2. 所有测试都通过
3. 提供了足够的文档说明
4. 包含了相关的测试用例
5. PR描述清楚地说明了更改内容和目的

## 许可证

通过贡献代码，您同意您的贡献将根据项目的GPL v3.0许可证进行许可。
